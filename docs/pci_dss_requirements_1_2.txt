PCI-DSS Requirements 1 and 2 — Network Security and Secure Configuration

Requirement 1.1 — Firewall Configuration Standards
All organizations must establish firewall configuration standards that include a formal process for approving and testing all network connections and changes to the firewall and router configurations. Firewall rules must be reviewed at least every six months. All unnecessary connections must be denied by default.

Requirement 1.2 — Restrict Connections
Firewalls must be configured to restrict inbound and outbound traffic to only that which is necessary for the cardholder data environment. All other traffic must be explicitly denied. Connections between untrusted networks and any system component in the cardholder data environment must be restricted.

Requirement 1.3 — Network Access Controls
Direct public access between the Internet and any system component in the cardholder data environment must be prohibited. A DMZ must be implemented to limit inbound traffic to only system components that provide authorized publicly accessible services. All other inbound and outbound traffic must be denied.

Requirement 2.1 — Default Passwords
Vendor-supplied defaults must always be changed before installing a system on the network. This includes passwords, SNMP community strings, and any other vendor default settings. Default accounts that are not required must be removed or disabled.

Requirement 2.2 — System Configuration Standards
Configuration standards must be developed for all system components. These standards must address all known security vulnerabilities and must be consistent with industry-accepted system hardening standards such as CIS Benchmarks or NIST guidelines.

Requirement 2.3 — Encryption of Non-Console Access
All non-console administrative access must use strong cryptography. Web-based management interfaces must use TLS 1.2 or higher. Telnet and other insecure protocols must never be used for administrative access.
